# v0.1 Scope

Version 0.1A builds the local service foundation, registries, safety policy, project inspection, Codex CLI detection, sample project, and documentation.

Version 0.1B workflow foundation adds task queues, event logs, approval queues, project write locks, dry-run planning, action receipts, risk budget checks, diagnostics export, and implementation report validation.

The Codex planning slice adds future-run plan records, command previews, prompt planning, approval integration, and diagnostics summaries. It does not execute Codex.

The controlled execution slice adds a single approved-plan Codex CLI execution endpoint. It does not add generic shell execution, unrestricted review loops, or arbitrary check runners.

The post-Codex review slice adds a policy gate after approved Codex execution. It inspects git status and diff stats, enforces changed-file and diff-line budgets, flags protected-file and dependency/package-file path changes, and only plans checks from detected package scripts.

The check execution receipts slice runs only that generated safe check plan, records Safe Action Runtime receipts and redacted results, and stops on the first failed or blocked check.

The controlled repair loop slice may run at most two Codex repair attempts after failed safe checks. It reruns post-Codex review and safe checks after each repair and stops on repeated failures, review-required changes, protected/dependency changes, risk-budget issues, or Codex repair failure.

## Not Included

Version 0.1B still excludes generic Codex execution, unrestricted repair loops, full review loops, dashboards, LAN pairing, desktop shells, installers, YouTube, health, Gmail, calendar, social, business, finance, cloud sync, mobile apps, payments, external account connectors, and browser automation.
